# Rubric Flip Framework

> Originally surfaced in [[Beliefs/Schooling Rubrics are out dated]]. The education argument is the clearest version, but the principle applies everywhere.

---

## The Core Principle

> **What you reward is what you get. If your system measures the wrong output, people will optimize for the wrong output.**

This is not a motivational observation. It's a systems diagnosis.

When a system produces bad output consistently, the instinct is to blame the people. The real answer is almost always: **you're measuring the wrong thing.**

---

## The Pattern (in any context)

| Old metric (what the system rewards) | Real metric (what actually matters) |
|--------------------------------------|--------------------------------------|
| Output compliance | Output quality |
| Quantity | Signal-to-noise ratio |
| Looking right | Working right |
| Process followed | Problem actually solved |
| Activity | Leverage |

---

## Where This Shows Up in Client Work

**CRMs nobody uses**
The system rewards "having a CRM." Nobody measures adoption, clean data, or whether follow-up actually happens. Result: beautiful CRM, zero behavior change.

**Funnels that look good but don't convert**
The system rewards "launching a funnel." Nobody measures message match, trust signals, or friction points. Result: clean design, no leads.

**SOPs that get ignored**
The system rewards "documenting the process." Nobody measures whether the team can actually run it without you. Result: documentation theater.

**Content that gets posted but doesn't build**
The system rewards posting frequency. Nobody measures whether each piece builds trust or moves a prospect closer. Result: content calendar full, pipeline empty.

---

## The Flip (how to apply it)

When diagnosing a client system, ask:
1. What does this system currently reward?
2. Is that the thing that actually matters?
3. What would the metric look like if we flipped it?

If the answer to #2 is no — that's the bottleneck. Not the people. Not the tools. The rubric.
