## My Take
Apple Notes for capture is the right call — zero friction, already open, no setup tax.
Trying to make Obsidian mobile do double duty would've killed the habit before it started.
The export skill is what makes the whole thing work — without it, notes either get sanitized
into useless summaries or stay raw and unsearchable. This is the system I actually need.

## The Decision
- Use Apple Notes for all mobile capture — no Obsidian mobile
- Weekly batch transfer: Apple Notes → Obsidian daily notes on desktop
- Use the obsidian-export skill at the end of every Claude brainstorm session
- Structured note format with 3 templates: Strategy, Debrief, Idea Dump

## Why This Way
Obsidian mobile adds sync setup and plugin compatibility friction that doesn't serve
the capture phase at all. The synthesis and connection work happens on desktop anyway.
Separating capture from structure is cleaner — Apple Notes handles the former,
Obsidian and the export skill handle the latter.

## Key Conclusions
- Apple Notes = capture layer. Obsidian = synthesis layer. Never conflate the two.
- The export skill self-detects entities and session type — no hardcoded client lists
- Raw Thinking stays stream of consciousness — no sanitizing
- Vault Update Suggestions are always labeled as inferred, not instructions
- Weekly transfer is the discipline dependency — skill can't fix that

## Open Questions / Next Moves
- Set a recurring weekly reminder to do the Apple Notes → Obsidian transfer (Sunday?)
- Test the export skill on a real client brainstorm session end-to-end
- Install the skill via Settings → Custom Skills
