# Claude Code — Production Layer

> Claude Code is where serious work happens. I use it to build complex systems, refactor Lovable prototypes into production-ready code, and tackle anything that requires real engineering judgment.

---

## Role in My Stack

**Claude Code = depth, quality, and real builds.**

Where Lovable moves fast and rough, Claude Code moves deliberately and properly. It's the environment I use when I'm building something that needs to actually work — for clients, for users, or for myself long-term.

---

## What I Use Claude Code For

- Refactoring and hardening Lovable prototypes
- Building new complex features from scratch
- Client site builds (from design to deploy-ready code)
- Automation tools, MVPs, and internal systems
- Revisiting and redesigning old projects
- Systems thinking — architecture, data flow, tool integrations

## What I Do NOT Use Claude Code For

- Quick throwaway prototypes (that's Lovable)
- Pure ideation without a concrete goal (that's Obsidian)

---

## How I Work With Claude Code

### Starting a Session
Before diving in, I write a brief project note in Obsidian with:
- What I'm building / what problem I'm solving
- Current state (new build? refactor? redesign?)
- Key decisions already made
- What "done" looks like for this session

Then I paste that context into Claude Code to start the session with full context.

### During a Session
- Let Claude Code drive the implementation
- Review and push back when something feels off
- Make architectural decisions together — don't just accept the first approach
- Take note of interesting decisions or tradeoffs

### After a Session
- Log key decisions and outcomes back into the project note in Obsidian
- Note what's next so the next session starts warm
- If something didn't work, write down why

---

## Obsidian as the Documentation Layer

Every meaningful Claude Code project has a corresponding Obsidian project note. This note is:
- The brief before work starts
- The decision log during work
- The record of what was built and why after

This keeps context portable — I can hand any project note to Claude Code and it picks up immediately.
