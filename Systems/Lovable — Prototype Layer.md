# Lovable — Prototype Layer

> Lovable is my fast-draft environment. I use it to generate rough UIs, test layout ideas, and get something visual on screen quickly — not to ship production code.

---

## Role in My Stack

**Lovable = speed and shape, not quality.**

When I have an idea for a UI, flow, or site, Lovable lets me generate a rough version in minutes. That rough version then goes into review: what's good, what's wrong, what needs to be rebuilt properly in Claude Code.

Lovable output is always a **draft**. It never ships directly.

---

## What I Use Lovable For

- First-pass layouts for client sites and landing pages
- Quick UI mockups to validate an idea before investing real build time
- Generating rough component structures to hand off to Claude Code
- Exploring design directions (layout, color, copy flow)

## What I Do NOT Use Lovable For

- Final production builds
- Complex logic or dynamic behavior
- Anything a real user will interact with before Claude Code review

---

## The Handoff Rule

> If I'd be embarrassed to show a client the Lovable output as-is, it goes to Claude Code before anyone sees it.

Every Lovable project that advances gets:
1. A review note in Obsidian (what to keep, what to fix, what to scrap)
2. A Claude Code session to harden or rebuild it
